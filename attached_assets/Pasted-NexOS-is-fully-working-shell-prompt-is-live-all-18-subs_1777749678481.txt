PHASE 1 — VIRTUAL DISK (FAT32 on VDI)
════════════════════════════════════════════════════════

Goal: Create a virtual disk image that VirtualBox can attach 
as a hard drive, pre-formatted as FAT32, so NexOS auto-mounts 
it at /mnt/disk on boot and the shell can read/write files 
that persist across reboots.

1A — Create disk image build script
Create tools/create_disk.sh:

  #!/bin/bash
  # Creates a 64MB FAT32 disk image for NexOS
  set -e
  IMG=build/nexos_disk.img
  MOUNT=/tmp/nexos_disk_mount
  
  echo "[*] Creating 64MB disk image..."
  dd if=/dev/zero of=$IMG bs=1M count=64
  
  echo "[*] Formatting as FAT32..."
  mkfs.fat -F 32 -n "NEXOS" $IMG
  
  echo "[*] Mounting and populating..."
  mkdir -p $MOUNT
  sudo mount -o loop $IMG $MOUNT
  
  mkdir -p $MOUNT/home/user
  mkdir -p $MOUNT/var/log
  mkdir -p $MOUNT/tmp
  echo "NexOS persistent disk" > $MOUNT/README.txt
  echo "hostname=nexos" > $MOUNT/nexos.conf
  
  sudo umount $MOUNT
  rmdir $MOUNT
  echo "[*] Done: $IMG ($(du -h $IMG | cut -f1))"

1B — Add make disk target to Makefile:
  disk:
      bash tools/create_disk.sh

  run-with-disk:
      make iso

1C — Fix FAT32 BPB detection in kernel/fs/fat32.c
The current fat32_mount() fails with "BPB invalid" because 
it only checks the boot signature (0x55AA) but not the OEM 
string or cluster count. Fix fat32_mount() validation:

  a) Read sector 0 from the ATA drive into a 512-byte buffer
  b) Check bytes 510-511 == 0x55, 0xAA (boot signature)
  c) Read the BPB:
     - bytes_per_sector at offset 11 (must be 512)
     - sectors_per_cluster at offset 13 (must be power of 2)
     - num_fats at offset 16 (must be 1 or 2)
     - total_sectors_32 at offset 32
     - fat_size_32 at offset 36
     - root_cluster at offset 44
  d) Verify: bytes_per_sector == 512 AND num_fats >= 1
  e) Log each field at DEBUG level so we can see what's read:
     klog(DEBUG, "FAT32 BPB: bps=%d spc=%d nfats=%d",
          bpb->bytes_per_sector,
          bpb->sectors_per_cluster,
          bpb->num_fats);
  f) Only return error if signature AND bps AND num_fats all fail

PHASE 3 — NETWORKING (RTL8139 driver)
════════════════════════════════════════════════════════

Goal: Detect and initialize an RTL8139 network card in QEMU 
(-netdev user,id=n0 -device rtl8139,netdev=n0) and implement 
enough to send/receive raw Ethernet frames.

3A — RTL8139 PCI detection (kernel/drivers/rtl8139.c)
Create kernel/drivers/rtl8139.c and kernel/drivers/rtl8139.h

  RTL8139 identifiers:
    Vendor ID: 0x10EC
    Device ID: 0x8139

  In rtl8139_init():
    a) Scan the PCI device list (already built by pci_init())
       for vendor=0x10EC device=0x8139
    b) If not found, log INFO "RTL8139: not found" and return 0
    c) If found, read the BAR0 (I/O base address) from PCI 
       config space
    d) Enable PCI bus mastering (set bit 2 in Command register)
    e) Power on: write 0x00 to CONFIG_1 register (offset 0x52)
    f) Software reset: write 0x10 to CMD register (offset 0x37)
       Poll until bit 4 clears (reset complete), timeout 1000ms
    g) Read MAC address from registers 0x00-0x05:
       uint8_t mac[6];
       for (int i=0; i<6; i++) mac[i] = inb(iobase + i);
    h) Log: "[INFO] RTL8139: found at PCI, MAC=XX:XX:XX:XX:XX:XX"

3B — Receive buffers
    a) Allocate a 32KB + 16 byte receive ring buffer using kmalloc
    b) Write physical address to RBSTART register (offset 0x30)
    c) Set IMR (offset 0x3C) = 0x0005 (ROK + TOK interrupts)
    d) Set RCR (offset 0x44) = 0xF | (1<<7)  (accept all, wrap)
    e) Enable RX+TX: write 0x0C to CMD register (offset 0x37)

3C — Transmit function
  int rtl8139_send(uint8_t *data, uint16_t len):
    a) Use one of 4 TX descriptors (rotate with tx_slot % 4)
    b) Copy data to tx_buffer[slot] (each 2KB)
    c) Write physical address to TSAD[slot] (offset 0x20+slot*4)
    d) Write len to TSD[slot] (offset 0x10+slot*4) with bit 13 
       clear (starts transmit)
    e) Poll TSD[slot] bit 15 (TOK) for completion, timeout 100ms
    f) Return 0 on success, -1 on timeout

3D — IRQ handler
    a) Install RTL8139 on IRQ 11 (typical for PCI slot in QEMU)
       or read the interrupt line from PCI config offset 0x3C
    b) In IRQ handler: read ISR register (offset 0x3E)
    c) If ROK (bit 0): call rtl8139_receive() to drain RX buffer
    d) If TOK (bit 2): mark last TX as complete
    e) Write ISR value back to clear interrupts

3E — Receive function
  void rtl8139_receive():
    a) Read CAPR register (offset 0x38) for current read pointer
    b) Read CBR register (offset 0x3A) for current buffer pointer
    c) While CAPR != CBR:
       - Read 4-byte packet header at rx_buffer + CAPR
       - Extract packet length from bits 16-31 of header
       - Pass packet data to a rx_callback function pointer
       - Advance CAPR by packet_length + 4, 4-byte aligned
       - Write new CAPR to register

3F — Simple ping handler (ARP reply)
  Implement a basic ARP responder:
    a) When a packet arrives, check EtherType (offset 12-13)
    b) If EtherType == 0x0806 (ARP):
       - Check ARP operation == 1 (request)
       - Check target IP matches our IP (use 10.0.2.15 for QEMU NAT)
       - Build ARP reply: swap src/dst MAC and IP, set op=2
       - Call rtl8139_send() with the reply
    c) Log: "RTL8139: ARP reply sent to X.X.X.X"

3G — Shell command: ifconfig
  Add to nsh.c:
  "ifconfig" command prints:
    "eth0: MAC=XX:XX:XX:XX:XX:XX  IP=10.0.2.15"
    "      RX packets: N   TX packets: N"
    "      Status: UP"

3H — Shell command: ping [ip]
  Add to nsh.c:
  "ping [ip]" sends 4 ICMP echo requests:
    - Build Ethernet + IP + ICMP headers manually
    - IP: src=10.0.2.15, dst=[ip], proto=1 (ICMP)
    - ICMP: type=8 (echo request), code=0, seq=1-4
    - Compute IP and ICMP checksums
    - Send via rtl8139_send()
    - Wait 1 second for reply via rx callback
    - Print: "64 bytes from [ip]: icmp_seq=1 time=Xms"
    - Or: "Request timeout for icmp_seq=1"

3I — QEMU networking flag
  Update tools/run_qemu.sh to add:
    -netdev user,id=n0 \
    -device rtl8139,netdev=n0 \
  So the network card is present when testing.

3J — Call rtl8139_init() from kernel_main()
  In kernel/kernel.c, after pci_init(), add:
    rtl8139_init();
  Add #include "drivers/rtl8139.h" at the top.
