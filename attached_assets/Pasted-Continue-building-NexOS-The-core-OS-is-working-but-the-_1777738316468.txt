TASK 1 — VMM (Virtual Memory Manager)
════════════════════════════════════════
Create kernel/mm/vmm.c and kernel/mm/vmm.h

Implement:
- vmm_map_page(phys_addr, virt_addr, flags) — maps a physical page to a 
  virtual address in the current PML4 by walking and allocating page table 
  entries as needed
- vmm_unmap_page(virt_addr) — removes a mapping and flushes TLB via invlpg
- vmm_get_physical(virt_addr) — walks the page table and returns the 
  physical address mapped to a given virtual address, or 0 if not mapped
- vmm_create_address_space() — allocates a new PML4 table, copies kernel 
  mappings into the upper half, returns the physical address of the new PML4
- vmm_switch_address_space(pml4_phys) — writes pml4_phys to CR3

Integrate into kernel.c init sequence between PMM and Heap. Add a klog 
INFO message on success.

TASK 2 — Ring 3 User Mode Execution
════════════════════════════════════════
Verify and fix process.c and scheduler.c so that user-space processes 
(init and nsh) actually run in CPU privilege ring 3, not ring 0.

Requirements:
- Each user process must have its own user-mode stack mapped in its virtual 
  address space (e.g. at 0x00007FFFFFFFE000)
- The first switch to user mode must use IRET with:
    * CS = user code segment selector | 3 (RPL=3)
    * SS = user data segment selector | 3
    * RFLAGS with IF=1 (interrupts enabled)
    * RIP pointing to the process entry point
    * RSP pointing to the user-mode stack top
- The TSS must hold the kernel RSP0 for each process so syscalls via 
  INT 0x80 correctly switch to the kernel stack
- Syscall return must IRET back to ring 3 correctly
- Test: kernel_main() should log "Dropping to ring 3 — launching init" 
  just before the first user process switch

TASK 3 — Shell Pipes
════════════════════════════════════════
Add pipe support to userspace/shell/nsh.c

Implement: cmd1 | cmd2

- When the shell parser finds a | character, split the command line into 
  left and right halves
- Create a pipe: an in-memory circular buffer (2048 bytes) in the shell's 
  heap, with a read fd and write fd assigned in the fd table
- Fork left command: redirect its stdout (fd 1) to the write end of the pipe
- Fork right command: redirect its stdin (fd 0) to the read end of the pipe
- Shell waits for both child processes to complete before showing the prompt
- Supported example: cat /etc/motd | cat (at minimum — real piping between 
  any two commands)
- Limit: support a single pipe only (no chaining cmd1 | cmd2 | cmd3)

TASK 4 — Shell Output Redirection
════════════════════════════════════════
Add output redirection to userspace/shell/nsh.c

Implement: cmd > file  and  cmd >> file

- Parser: after splitting into tokens, check for > or >> tokens
- > file: open the file with O_CREATE | O_WRONLY | O_TRUNC, dup the fd to 
  stdout (fd 1) before executing the command
- >> file: same but open with O_APPEND instead of O_TRUNC
- After the command finishes, restore original stdout fd
- Example: echo hello > /home/user/test.txt

TASK 5 — Shell Background Jobs
════════════════════════════════════════
Add background execution to userspace/shell/nsh.c

Implement: cmd &

- If the last token is &, remove it and fork the command without waiting 
  for it to finish
- Print: [1] <pid> to the terminal after forking
- Shell immediately returns to the prompt
- When the background job finishes, the next time the shell reads input, 
  check for zombie children via sys_wait with WNOHANG flag and print:
  [1]+  Done    <command>

TASK 7 — /proc Virtual Filesystem
════════════════════════════════════════
Implement a read-only /proc filesystem backed by VFS.

Create kernel/fs/procfs.c and kernel/fs/procfs.h

Mount at /proc during kernel init (after scheduler is running).

Provide these virtual files:
- /proc/version     — "NexOS 0.1 built [date]"
- /proc/uptime      — uptime in seconds (from PIT ticks / 1000)
- /proc/meminfo     — total physical memory, free pages, heap used/free
- /proc/<pid>/      — directory per running process
- /proc/<pid>/status — PID, name, state, parent PID
